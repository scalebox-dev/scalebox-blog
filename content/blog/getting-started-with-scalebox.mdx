## Overview

ScaleBox makes it easy to create isolated cloud sandboxes for development, testing, and CI/CD workflows. In this post, we'll walk through the basics of setting up your first sandbox environment.

## Prerequisites

Before getting started, make sure you have:

- A ScaleBox account with an API key
- Node.js 18+ or Python 3.9+ installed
- The ScaleBox CLI or one of the official SDKs

## Creating Your First Sandbox

### Using the CLI

The fastest way to get started is with the ScaleBox CLI:

```bash
# Install the CLI
npm install -g @scalebox/cli

# Authenticate
scalebox auth login

# Create a sandbox from a template
scalebox sandbox create --template ubuntu-base
```

### Using the JavaScript SDK

For programmatic access, use the JS SDK:

```typescript
import { ScaleBox } from '@scalebox/sdk';

const client = new ScaleBox({ apiKey: process.env.SBX_API_KEY });

const sandbox = await client.sandbox.create({
  template: 'ubuntu-base',
});

console.log(`Sandbox created: ${sandbox.id}`);
```

### Using the Python SDK

Or if you prefer Python:

```python
from scalebox import Sandbox

sandbox = Sandbox.create(template="ubuntu-base")
print(f"Sandbox created: {sandbox.id}")
```

## What's Next

Now that you have a running sandbox, you can:

- **Execute commands** — Run shell commands inside the sandbox
- **Transfer files** — Upload and download files to/from the sandbox
- **Use Code Interpreter** — Run code in multiple languages with built-in kernels
- **Connect via WebSocket** — Get a real-time terminal session

Check out our [Guides](https://docs.scalebox.dev/en/guides) and [API Reference](https://docs.scalebox.dev/en/api) for more details.
